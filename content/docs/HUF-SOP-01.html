1 Purpose & Scope

This procedure ensures use-related risks are identified and controlled and that the user interface is safe and effective for the intended users, tasks and use environments.

Scope: usability engineering for all devices with a user interface. It satisfies IEC 62366-1 and feeds the risk file (RSK-SOP-01) and design & development (DEV-SOP-03).

2 Roles & Responsibilities

3.1 Use specification

InputIntended use; users; environment
Activity• Define the intended users, use environments and the user interface.
• Identify the operational/use scenarios, including the primary operating functions.
• Record the use specification in the Usability Engineering File.
ResponsibleUsability Engineer (R)
Method / CriteriaIEC 62366-1 §5.1-5.3
Tools / TemplateHUF-TPL-01 UEF
Output / RecordUse specification
ClauseIEC 62366-1 §5.1

3.2 Identify use-related hazards

InputUse specification
Activity• Identify hazard-related use scenarios and potential use errors.
• Derive the critical tasks whose incorrect performance could cause harm.
• Feed these into the risk file.
ResponsibleUsability Engineer (R); Risk Manager (Support)
Method / CriteriaIEC 62366-1 §5.4-5.5
Tools / TemplateRSK-TPL-03
Output / RecordHazard-related use scenarios
ClauseIEC 62366-1 §5.5

3.3 User interface specification & evaluation plan

InputCritical tasks
Activity• Specify the user interface requirements addressing the critical tasks.
• Prepare the usability evaluation plan defining formative and summative methods and acceptance criteria.
ResponsibleUsability Engineer (R)
Method / CriteriaIEC 62366-1 §5.6-5.8
Tools / TemplateHUF-TPL-02 Evaluation Plan
Output / RecordUI specification; evaluation plan
ClauseIEC 62366-1 §5.8

3.4 Formative evaluation

InputUI prototype
Activity• Conduct formative evaluations during design to find and fix use problems early.
• Feed findings back into the UI design and risk controls.
ResponsibleUsability Engineer (R)
Method / CriteriaIEC 62366-1 §5.9 (formative)
Tools / TemplateHUF-TPL-03 Evaluation Protocol
Output / RecordFormative findings
ClauseIEC 62366-1 §5.9

3.5 Summative evaluation

InputFinal UI
Activity• Conduct the summative evaluation of the critical tasks under representative conditions.
• Demonstrate that use-related risk is acceptable and document residual use errors.
ResponsibleUsability Engineer (R)
Method / CriteriaIEC 62366-1 §5.9 (summative)
Tools / TemplateHUF-TPL-04 Summary Report
Output / RecordSummative results
ClauseIEC 62366-1 §5.9

3.6 Conclude the UEF

InputEvaluation results
Activity• Conclude on use safety and compile the Usability Engineering File.
• Confirm residual use-related risk is acceptable and feed the conclusion to the risk file and clinical evaluation.
ResponsibleUsability Engineer (R)
Method / CriteriaIEC 62366-1
Tools / TemplateHUF-TPL-01 UEF; RSK-TPL-03
Output / RecordUse-safety conclusion
ClauseIEC 62366-1
