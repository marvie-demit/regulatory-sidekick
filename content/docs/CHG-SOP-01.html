1 Purpose & Scope

This procedure ensures changes to products, software, processes and the management system are assessed for impact, approved before implementation, implemented in a controlled way and traceable.

Scope: all planned changes (design, software/deployment, process, supplier, document, AI model). It satisfies ISO 13485 4.1.4 & 7.3.9, MDR Art 10(9), ISO 27001 cl.8.1 and ISO 42001 cl.8.1. Emergency changes follow the same steps retrospectively.

3 Procedure

Guidance (delete before release): Each step is a structured configuration (IDEF0/ICOM): Input, Activity, Responsible (RACI), Method/Criteria (Control), Tools/Template (Mechanism), Output/Record, Clause.

3.1 Raise change request

InputChange need (improvement, CAPA, regulatory, supplier, design/model)
Activity• Submit a change request describing the change, its rationale and its scope.
• Log the request in the Change List before any implementation.
ResponsibleChange Initiator (R); QMR (A)
Method / CriteriaAll changes logged before implementation
Tools / TemplateCHG-FOR-01; CHG-LIS-01
Output / RecordLogged change request
ClauseISO 13485 4.1.4 / 7.3.9

3.2 Impact & significance assessment

InputChange request
Activity• Assess the impact on safety, performance, security and validation.
• Identify affected documents, risk files and downstream processes.
• Determine the regulatory significance (e.g. MDR significant change) and notification needs.
• For AI, determine whether the change is within the Predetermined Change Control Plan envelope (AIM-TPL-04).
ResponsibleChange Review Board (R); RA (Support)
Method / CriteriaDocumented impact assessment; significance per MDCG guidance
Tools / TemplateCHG-FOR-01
Output / RecordImpact assessment
ClauseMDR Art 10(9)

3.3 Approve or reject

InputImpact assessment
Activity• Decide to approve or reject the change.
• Define the required actions, verification/validation and acceptance criteria.
• Escalate significant changes to Top Management.
ResponsibleChange Review Board (R); Top Management (A - significant)
Method / CriteriaDual review; significant changes escalated
Tools / TemplateCHG-FOR-01
Output / RecordDecision
ClauseISO 13485 7.3.9

3.4 Implement

InputApproved change
Activity• Execute the change.
• Update documents, risk files and software/deployment.
• Perform the required verification and validation.
ResponsibleProcess Owner (R)
Method / CriteriaPer approved plan & acceptance criteria
Tools / TemplateDEP-SOP-01; RSK-SOP-01
Output / RecordImplemented change
ClauseISO 13485 7.5

3.5 Verify & notify

InputImplemented change
Activity• Verify the change is effective and the acceptance criteria are met.
• Notify the Notified Body / competent authority where the change is significant, before release.
ResponsibleQMR (R); RA (Support)
Method / CriteriaNotification where significance criteria met
Tools / TemplateCHG-LIS-01
Output / RecordVerification; notification
ClauseMDR Art 10(9)

3.6 Close

InputVerified change
Activity• Confirm all actions are complete.
• Close the change in the Change List and release.
ResponsibleQMR (R)
Method / CriteriaAll actions complete
Tools / TemplateCHG-LIS-01
Output / RecordClosed change
ClauseISO 13485 4.2.5
